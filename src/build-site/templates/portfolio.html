  • My talk for Guadalindie

    This talk for Guadalindie is titled "QA for indies". It's a primer on QA basics which aims to improve the quality of your project without dedicating a lot of resources. We can do this by focusing on 3 main topics: communication, testing and automation. More resources and links can be found in its companion blog post.


  • This blog post about configuring a simple CI environment

    I wrote a blog post about configuring a simple continuous integration (CI) environment. By following this article you can end up automating the build and publishing your Godot game on Itch.io in an hour or less! This introduces people to concepts like Continuous Integration, automation, build configurations, API tokens and shell scripting.

    Barcelona Game Creators is a monthly meetup where local gamedevs can give short presentations to share learnings/progress and meet other gamedevs and network.

GATO (Godot Accessibility Toolkit)

This is a (yet to be released) collection of demos, documentation and plugins that aims to make implementing accessibility features easier for Godot games.

In this project I'm the lead programmer. We've had 2 people learning programming with us and I've mentored them, doing pair programming and teaching them concepts they need to know to work on the game.

🌐 Website

Michiball

This was the first project we started as Nokorpo, after the jams. We wanted a small game we could finish in some months and publish in Steam quickly, since our goal was to learn as much as possible.

Since this is our first big project, we worked together early on to define our standards: how we would write and architect code, how we work as a team, our responsibilities... This was also when we started writing automated tests for our games.

La causa del accidente que provocó el incendio

This is a project we made in Nokorpo for ICEX (the Spanish Institute for Foreign Trade). They made a campaign to promote audiovisual talent from Spain, which included a short movie and the short game we made.

I wrote most of the code for this project and focused on writing tools to help our game designer to iterate fast.

Infinigrass

A jam game made in 3 days for Mermelada Jam 2024. Achieved 1st 🥇 place in the "Art" and "Fun" categories. An incremental design in a very short experience (10 minutes).

I made the game and sound design for this one, as well as coding.

🎮️ Play it here!

Rat and Furrius

Anoter jam game made in 3 days for Mermelada Jam 2023. Achieved 2nd place 🥈 in the "Innovation" and "Fun" categories. Very quick and engaging gameplay.

For this game I mostly worked in programming and production. Also worked very closely with the game designer to prepare the tools they needed as well as helping to iterate quickly during polishing.

🎮️ Play it here!

GDSnap

This is a tool for enabling Screenshot Testing in Godot. I made it as a proof of concept and shared it in the Guadalindie talk. The maintainers for some testing plugins have gotten in contact with us to integrate it with their plugins, so it will soon have more development.
